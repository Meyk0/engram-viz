Install the CLI as a development dependency:

```bash
npm install --save-dev @engramviz/cli
```

| Command | Description |
| --- | --- |
| `engram init [--project name]` | Create local project identity and capture credentials |
| `engram dev [--port 3100]` | Start the packaged Engram Studio |
| `engram env [--format shell\|json]` | Print agent-safe capture environment values |
| `engram run [--port 3100] -- <command>` | Run a child process with capture configured |
| `engram doctor [--port 3100]` | Check Node, project configuration, and Studio availability |
| `engram import <capture.json>` | Import an `engram.capture` bundle |
| `engram test <artifact> --executor <module>` | Execute a regression through project-owned code |
| `engram test <artifact> --observation <json>` | Validate an existing observation |

## `engram init`

Creates `.engram/config.json` and adds `.engram/config.json` plus `.engram/data/` to the repository's `.gitignore`.

```bash
npx engram init --project support-agent
```

Running `init` again preserves the existing token and project identity.

## `engram dev`

Starts Studio at `http://localhost:3100/?mode=incidents` by default:

```bash
npx engram dev
npx engram dev --port 3200
```

## `engram env`

Use this when a process manager needs the capture values without `engram run`:

```bash
npx engram env --format shell
npx engram env --format json
```

The output contains only `ENGRAM_URL`, `ENGRAM_TOKEN`, and `ENGRAM_PROJECT_ID`.

## `engram run`

Everything after `--` is executed as the agent command:

```bash
npx engram run -- npm run dev:agent
npx engram run -- node scripts/reproduce-incident.mjs
```

The child process inherits its normal environment plus the Engram capture values.
